## Technical Details

### Ray Manager RBAC Configuration
```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: ray-manager
  namespace: default
---
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: ray-manager
  namespace: default
rules:
- apiGroups: ["ray.io"]
  resources: ["rayclusters", "rayjobs"]
  verbs: ["create","get","list","watch","update","patch","delete"]
```

### Cluster CRD Specification
```protobuf
message ClusterSpec {
  enum SLAType {
    SLA_TYPE_INVALID = 0;
    SLA_TYPE_PRODUCTION = 1;
    SLA_TYPE_EPHEMERAL = 2;
  }

  SLAType sla = 1;
  string region = 2;
  string zone = 3;

  oneof cluster {
    option (michelangelo.api.required) = true;
    KubernetesSpec kubernetes = 5;
  }

  DCType dc = 7;
}
```

### Development Setup
For local development:
1. Install required tools:
   ```bash
   # Install Ray operator
   kubectl create -k github.com/ray-project/kuberay/ray-operator/config/default?ref=v1.0.0

   # Apply RBAC configuration
   kubectl apply -f rbac-ray.yaml
   ```

2. Configure IDE:
   - Use VSCode/Cursor with gopackagesdriver
   - Enable TypeScript and Python language servers
   - Configure linting tools (ruff, eslint)
